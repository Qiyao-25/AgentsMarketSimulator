# AI Agent 金融市场模拟大赛 初赛

## 1. 赛题引入：当算法遇见人性

在金融领域的深水区，传统模型往往陷入“理性人”的荒漠：它们能算准波动率，却算不准人心。真实的股市不仅是数字的跳动，更是成千上万个灵魂在社交媒体上的讨论、在恐慌中的踩踏、在贪婪里的博弈。

本次竞赛邀请你利用大语言模型（LLM）的认知能力，构建一套具备 BDI（信念-欲望-意图）架构的智能代理（Agent）和交易基础设施。你的任务是：证明 AI Agent 不仅能执行规则，还能在新闻、社交信息和持仓盈亏共同作用下，表现出接近真实投资者的行为偏差，并协助交易所识别异常交易。

---

## 2. 比赛题目

### 题目一：灵魂的塑造（单体投资 Agent 建模）

背景：在真实市场中，个体的决策源于感知。散户不会直接面对原始数据，而是面对喧嚣的新闻、充满偏见的评论和账户的盈亏。

核心任务：

- 实现 InvestmentAgent 类：该 Agent 需具备接收 K 线、新闻摘要及社交帖子的接口。
- 构建认知流水线：将碎片化的外部信息转化为内部“信念”（对资产价值的动态判断）。
- 性格化注入：通过规则、提示词或模型调用，使 Agent 表现出特定的行为风格，例如面对亏损时焦虑的“激进型”，或坚持长期判断的“价值型”。

评分重点：你的 Agent 表现得像“人”吗？评测会观察其交易动作、内心独白、信念分数和持仓处境是否一致，并通过处置效应、信念-行动相关性、换手率分布等指标进行量化。

### 题目二：秩序的基石（交易所基础设施 Agent 开发）

背景：交易所不仅是撮合工具，更是市场的守护者。它需要在极短时间窗口内，从混乱的订单流中辨别出秩序与阴谋。

核心任务：

- 开发撮合引擎：实现标准的限价订单簿（LOB），确保交易严格遵循价格优先与时间优先原则。
- 智能监控系统：在交易所中集成监管逻辑，自动识别并拦截 Wash Trading（洗盘/对倒）、Spoofing（虚假申报）、Pump and Dump（拉高抛售）等恶意操纵行为。
- 输出可解释干预：每条预警或拦截动作需要给出自洽的 thought，说明触发原因和干预逻辑。

评分重点：你的系统足够客观和敏锐吗？评测会通过撮合正确性、异常检测 F1-Score、响应延迟和误报情况综合判断。

## 3. 评判标准

### 3.1 投资 Agent 的行为拟人性（100 分）

#### 1. 处置效应定量（30 分）

考核目的：衡量 Agent 是否表现出“卖出盈利、持有亏损”的典型非理性特征。

计算指标：PGR（实现盈利比例）与 PLR（实现亏损比例）。

计算公式：

$$DE = PGR - PLR = \frac{Realized\_Gains}{Possible\_Gains} - \frac{Realized\_Losses}{Possible\_Losses}$$

评分逻辑：

- 满分区间：$DE \in [0.05, 0.15]$，符合真实散户统计特征。
- 扣分项：若 $DE \leq 0$，说明 Agent 过于机械或超理性；若 $DE > 0.3$，说明过度恐惧或过度止盈，得分按偏差比例衰减。

#### 2. 决策信念相关性（40 分）

考核目的：量化 Agent 的“内心独白”与“实际动作”的一致性。

计算过程：

1. 提取 Agent 决策时的 thought、belief_score 和 sentiment_class。
2. 记录实际交易方向：买入为 1，观望为 0，卖出为 -1。
3. 计算信念/情感与交易方向之间的 Spearman 秩相关系数。

评分逻辑：相关性越高，说明 Agent 的“想法”和“动作”越一致；若 Agent “想买却卖”或长期逻辑混乱，会明显扣分。

#### 3. 换手率分布相似度（30 分）

考核目的：防止 Agent 出现高频刷单或完全不交易的极端行为。

计算指标：每日换手率分布的 Wasserstein Distance（WD）。

计算逻辑：计算选手 Agent 的换手率序列与参考交易分布的 WD 距离。距离越小，得分越高。

### 3.2 交易所 Agent 的监管敏锐度（100 分）

#### 1. 撮合正确性（30 分）

考核目的：验证交易所是否严格执行限价订单簿规则。

重点检查：

- 买卖订单能否在价格交叉时成交。
- 同价位订单是否遵循时间优先。
- 部分成交后剩余数量是否正确留存在订单簿。
- close_prices、trades、accepted_order_ids、rejected_order_ids 是否结构完整。

#### 2. 异常行为识别能力（40 分）

考核目的：检测交易所 Agent 对市场操纵行为的捕获率。

测试集会包含正常订单流和异常订单流，异常类型包括：

- Wash Trading（洗盘/对倒）：同一实体或关联账户自买自卖。
- Spoofing（虚假申报）：远离盘口的大额订单诱导市场预期。
- Pump and Dump（拉高抛售）：多账户协同拉升后集中卖出。

评分公式：

$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

其中 Recall 代表能否抓到坏人，Precision 代表是否误伤好人。

#### 3. 监管介入时机（30 分）

考核目的：衡量监管的及时性。

计算指标：延迟响应间隔 $\Delta T$。

评分逻辑：

- 在攻击发生后的 3 个 Tick 内响应：高分。
- 超过 10 个 Tick 后才响应：低分或不计分。
- 若采取停牌、撤单拦截、账户冻结等干预手段，thought 需要解释清楚原因。

## 4. 提交与本地评测

提交目录必须包含 `submission.py`，并暴露：

```python
def create_submission(config: dict):
    return TeamSubmission(config)
```

返回对象必须实现：

```python
reset(seed, config=None)
decide(observation)
match_orders(orders, last_prices, tick)
```

本地接口校验：

```bash
python -m submission_interface.validator path/to/team_dir
```

专项本地评测：

```bash
python evaluate_submission.py path/to/team_dir
```

注意：本地评测脚本用于开发参考，正式成绩以隐藏测试集为准。
